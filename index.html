Other independent projects



Chess AI -> Play it here!



With very little HTML, CSS, and Javascript experience, I created a website where people can play local chess vs another person, an AI, or watch AI vs AI. Using a rule-based implementation (not ML), I wanted to see how strong I could make a chess AI. This was an opportunity to test my understanding of search algorithms, namely the minimax algorithm which I implemented from scratch along with several optimization techniques, such as alpha-beta pruning and move sorting. The AI is limited by the time-complexity. At a depth of 3 (~10,000 final positions), it takes .5 seconds, whereas at depth of 4 (~200,000 final positions), it takes 5-10 seconds. There were also lots of exceptions that were difficult to account for (e.g. castling, en-passant, pawn-promotion, etc.), but I managed to complete all of these (perhaps with the exception of pawn-promotion, which is still buggy with the AI!). Overall, the project was a great way to incorporate lots of CSC skills like recursion, DFS, time-complexity, web dev, and organizing 5,000 lines of code. I learned the importance of refactoring and incremental development.





Zipf's Law Simulator -> Try it out here!



This is a text analysis project I originally created in Java and then made into website using Javascript, HTML, and CSS. The project brings together my interest in Linguistics and Computer Science as it demonstrates Zipf's Law -- a famous statitstical relationship between word frequencies in texts. On the right is an example of running the program with "Harry Potter and the Sorcerer's Stone", showing that each word's frequency is inversely proportional to its rank in the frequenecy distribution.
